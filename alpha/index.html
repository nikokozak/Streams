Ticker Alpha

Install instructions, update behavior, and how to rollback (downgrade) to a previous version.

Updates

  • The alpha build uses Sparkle auto-updates.
  • The appcast lives at /Streams/appcast-alpha.xml.
  • Update payloads (the Ticker-YYYY.MM.patch.zip) are hosted on GitHub Releases and must be publicly downloadable.

Fresh install (first time)

  1. Download the latest alpha zip from GitHub Releases.
  2. Unzip it to get Ticker.app.
  3. Drag Ticker.app into /Applications.
  4. Launch it from /Applications (recommended for Sparkle updates).

Rollback / downgrade

If a new alpha has a regression, you can install an older release manually.

  1. Quit Ticker completely.
  2. Download the older zip you want from GitHub Releases (e.g. Ticker-2025.12.1.zip).
  3. Unzip it to get Ticker.app.
  4. In /Applications, rename the current app (optional): Ticker.app → Ticker.backup.app.
  5. Drag the older Ticker.app into /Applications.
  6. Launch it.

Note: Sparkle will offer to update again if the appcast contains a newer build. If you need to stay pinned temporarily, ignore the update prompt (or disable automatic update checks if the app exposes that setting).

Common problems

  • “App is damaged” / blocked by Gatekeeper: alpha builds should be signed + notarized. If you still see a block, ensure you launched the copy in /Applications and re-download the zip from Releases.
  • Update download fails: confirm the zip URL in appcast-alpha.xml is reachable in a private/incognito browser window.

If you’re helping test, please include your macOS version, the Ticker version, and steps to reproduce when reporting issues.
